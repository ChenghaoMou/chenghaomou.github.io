Selective Language Modeling

Source: Rho-1: Not All Tokens Are What You Need

Almost like a different version of Adaptive Computation, but instead allocating different compute for different input, it is restricting learning from only certain tokens.

The first step of SLM is to build a reference model that can recognize high-quality tokens from in the input. This can be achieved by curating a high-quality dataset and training a model on top of it. From such a model, a loss associated with a token in a sequence can be formalised as:

What this does is basically masking out unimportant tokens during the training based on the difference between the reference model and the regular model.

To the reference model, a high-quality token means small loss (since the model is expecting such tokens) and therefore a high excess loss (surprising to the regular model but not the reference model) indicates tokens that the regular model should better focus on.
